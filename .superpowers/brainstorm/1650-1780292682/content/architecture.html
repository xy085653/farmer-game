🏗️ 整体架构设计

事件驱动 + 服务层 — Godot 4.6 C#

核心架构层次

┌──────────────────────────────────────────────────┐
│ 🎮 表现层 (View/Scene) │
│ Player.gd / CropSprite / UI / TileMap │
│ ↓ 发送事件 ↑ 订阅事件 │
├──────────────────────────────────────────────────┤
│ 🔗 事件总线 (EventBus) — 单例 Autoload │
│ C# event Action / 所有系统零耦合通信 │
│ ↓ 调度 ↑ 监听 │
├──────────────────────────────────────────────────┤
│ ⚙️ 服务层 (Services) — 接口 + 实现 │
│ ITimeService / IFarmService / IInventoryService │
│ IEconomyService / IWorldService / ICraftService │
│ ↓ 读写 ↑ 通知 │
├──────────────────────────────────────────────────┤
│ 💾 数据层 (Data / Resource) │
│ CropData / SeasonData / ItemData / SaveData │
│ Godot Resource (.tres) + JSON 持久化 │
└──────────────────────────────────────────────────┘

📨 事件流示例：播种

// 玩家右键种子 → 点击耕地
① PlayerController
→
EventBus.Publish(new PlantSeedEvent(seedId, tilePos))
→ IFarmService.HandlePlantSeed
→ 验证耕地、扣除种子
→
EventBus.Publish(new SeedPlantedEvent(seedId, tilePos))
→ UI 更新背包 / 显示作物 / 播放动画
→
EventBus.Publish(new InventoryChangedEvent())
→ IEconomyService 完全不知道种田的细节
💡 这个架构的好处： 将来加"社交系统"或"钓鱼系统"时，只需要新写一个 ISocialService，注册到 ServiceRegistry，监听/发布事件即可——一行代码都不改现有系统。
